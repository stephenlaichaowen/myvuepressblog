# 切版練習

# 開始

切版有兩種策略 : 一種叫做 Desktop First, 一種叫做 Mobile First.

  • 什麼是 Desktop First 策略呢 ? 簡單來說就是先把桌上型的版面排好, 然後依序平板, 手機, 手錶由大而小的一種概念.

  • Desktop First 有甚麼優點呢 ? 首先是圖片的解析度, 由於我們先製作桌上型螢幕尺吋的網頁, 圖片選擇一定是最大最清楚的, 之後開發平板/手機等較小尺寸的螢幕時圖片只會縮小, 解析度 / 畫素不會失真, 就沒有圖片糊掉的問題. 這是 Desktop First 最明顯的優點.

  • 反之, Mobile First 策略就是說先排好手機的畫面, 然後依序向上製作平板, 桌機的畫面. 也就是先把最小螢幕尺寸的網頁開發出來, 然後在製作大螢幕的畫面.

  • 由於先從小尺寸螢幕開始排版, 圖片的選擇就比較不確定, 除非已經確定桌上型螢幕的尺寸, 不然通常會先選一張較大尺寸的圖片, 透過 width: 100% 滿足父元素的寬度. 當開發平板 / 桌上型電腦螢幕時圖片畫素被放大,造成糊掉的現象, 所以還要再選用較大尺寸的圖片, 這樣修修改改讓手續變得很繁複.

因為設計稿的尺寸是 1200x2553(px), 所以正好採用 Desktop First 策略, 至於小尺寸螢幕的調校, 就使用 chrome 內建的開發者工具縮放視窗再慢慢調整囉.

# 視覺元素

  • 我把這個頁面由上到下分成以下幾個區塊, 所有元素用 .wrap 包起來

    • header (導覽列)
      • .sec1 (左邊菜單), .sec2 (右邊菜單)
    • .hero
      • .title
      • .subtitle
      • .btn-group
    • main
      • .section1 (圖文)
      • .section2 (圖文)
      • .section3 (兩欄式圖文)
      • .section4 (三欄式卡片)
    • footer

  • 這個頁面的寬度 1200px, 所以我在 body 給 maw-width: 1200px, 讓頁面最大寬度保持在 1200px, 當頁面縮小內容會等比例縮放, 做到不用 media query 就達到響應式的功能.

  • 使用 margin: 0 auto 讓網頁水平置中

  • 字體選用 Noto Sans TC

  • 網頁主色系有四種顏色 :

    • dark-color
    • primay-color
    • secondary-color
    • bg-color

加入背景圖

  • background: url(hero-bg.png) no-repeat
  • background-position: center center


加入導覽列

導覽列使用 flexbox 排列, 因為背景圖中間有一個圖示, 為了要讓菜單水平排列左右間距一樣, .menu 內建立兩個 section, 每個 section 放兩個 menu item, 也就是 .menu 和 .sec 兩個父元素都用 display: flex 排列子元素.


菜單文字下面加入波浪

  • 開啟設計稿 (Adobe Xd) : 點選 圖示, 導出 PNG 圖檔

  • 加入 border-bottom-image :

background-image: url(bb.png);

加入響應式功能

  • 斷點選擇 800px : 因為視窗內縮快到 800px 時第二和第三個菜單選項太緊靠背景圖中間的圓形圖示, 所以用 media query 在斷點 800px 隱藏菜單, 同時顯示漢堡圖示


  • 點擊漢堡圖示展開菜單 : 這部分用 javascript 控制, menu.classList.toggle('toggle'), 點擊圖示切換菜單 (展開/隱藏)


加入 hero 內容

  • title
  • subtitle
  • 兩個按鈕

這部分排版比較容易, 三個元素水平置中, 再調上下間距即可
