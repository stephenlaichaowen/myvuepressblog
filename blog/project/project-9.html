# 使用 VuePress 建立個人網站

# 為什麼用 VuePress 呢?

網站開發工具有上百種, 在此就不列舉了, 因為真的太多了, 以知名的 WordPress 來比較, 我想 VuePress 的推出就是要和 WordPress 一較高下吧, 因為名字很像, 這是我的猜測. 至於為什麼選擇 VuePress 這個工具呢? 因為我訂閱的 youtube 頻道前一陣子傳來許多 VuePress 的影片, 這些影片製作人 / youtube 都是 VueJS 愛用者, 起初還不以為意, 後來看了一段影片後點燃我的興趣, 便依照影片教學建立專案和頁面. 我發現原來建立部落格可以這麼的輕鬆寫意. 立即愛上了 VuePress. 接著陸續建立幾個頁面, 看了幾個不同的示範教學視頻, 很快地就有了我的網站了. 其實我以前也有個人網站, 不過因為用開發網頁的方式寫網誌非常的不對勁, 因為寫網誌只是要寫內文, 上傳圖片 / 影片, 大致就這樣. 用寫程式的方法寫網誌讓我沒有動力持續下去, 所以很可惜, 不然記錄下許多生活的點點滴滴, 隨時可以回味, 是多麼有意思的一件事呀!

# Markdown VS HTML

VuePress 的特色之一是採用 markdown 作為頁面的主要語言 ( 副檔名 .md ). markdown 是一個輕量型語言, 語法簡短, 可以快速上手, 比起 html 標籤好用而且視覺上看起來很清爽.

markdown                                      html
-----------                                   -------------------
# 這是標題                                     <h1>這是標題</h1>

- 蘋果                                         <li>蘋果</li>
- 橘子                                         <li>橘子</li>
- 芭樂                                         <li>芭樂</li>>

[進入 Google](https://www.google.com.tw)       <a href="https://www.google.com.tw" target="_blank">進入 Google</a>

![](https://www.photo.com/myPic)               <img src="https://www.photo.com/myPic">

除了 markdown, VuePress 也允許在頁面寫 html, css, 以及 javascript, 讓開發者添加樣式及各種互動, 這就是框架強大之處.

# 自動產生代碼

VuePress 讓我們不需要寫落落長的標頭資訊, 通常我們寫 html 會把很多相關字眼塞進 meta 標籤內, 目的是為了讓搜尋引擎找得到的我們的網頁. 我們也不需要寫 html, head, body 等標籤, 也不用引入第三方程式庫, 這些都是很瑣碎繁雜的細節, VuePress 會自動生程這些代碼, 讓我們可以專注在內容寫作上, 真的是一大福音.

# 輕鬆產生導航欄, 側邊攔及搜尋欄

VuePress 讓我們輕鬆的製作出 navbar, sidebar, search bar 等功能, 在 config.js 填入頁面的路徑就會自動完成連結並且附上一個搜索欄, 方便我們尋找網頁, 這是很貼心的功能, 通常個人網誌內容都很繁多, 可以多達幾百甚至幾千的頁面, VuePress 讓我們不用寫程式就能使用搜索服務真的揪心肝...

# 除了靜態內容, 還可以使用組件

VuePress 是建立於 VueJS 之上的開發工具, 所以支援組件開發是很自然的事. 我們可以開發組件, 然後在頁面調用組件使用, 讓頁面不會只是純文字 / 圖片, 可以產生許多互動. 我做了幾個組件, 分別是圖片庫, 留言版, 登錄表格, 個人簡歷等需要和後端服務器 / 資料庫存取資料. 這是很強大也是我最喜歡的功能.

# 強大的 SEO, 解決搜尋引擎找不到頁面的問題

VuePress 是一個產出 SPA / Single Page Application / 單頁式應用程序的框架工具, 而 SPA 的特性是在第一次進入網站後瀏覽器會下載前端所有網頁, 之後瀏覽任何頁面都是在瀏覽器上執行, 服務器端完全沒有角色. 這個特性讓搜尋引擎在網路上找不到網頁. 以往要處理這個問題需要很複雜的技術程序, 而 VuePress 幫我們解決這個問題, 省下許多寶貴的時間.

# VuePress 專案結構

  • components 資料夾顧名思義存放的是組件 副檔名 .vue
  • config.s 存放整個網站的路由, 也就是所謂的 router.
  • dist 資料夾存放打包後要上傳服務器的檔案.
  • 其餘的都是靜態頁面 .md 檔案, 最後會被編譯成 html 檔案.

VuePress 編譯後待上傳到雲端服務器的檔案

以上整個專案架構, 看起來是不是很清爽, 簡單容易理解呢?

# 總結

VuePress 讓開發個人網頁變成一件愉快的事, 讓人會愛上寫網誌, 我大力推薦給大家. 根據官網說明, VuePress 還處於開發階段中, 也就是說以後會有更讓人驚豔的功能推出, 讓我們拭目以待吧...
