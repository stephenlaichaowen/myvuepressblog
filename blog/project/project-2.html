# 前言

我在大學主修電子工程, 所以對於微控制器的一切很熟悉, 包括韌體編程 / 燒錄, 電子元件焊接, 電源供應器, 示波器, 波形產生器等儀器的操作. 自從 Arduino 問世後, 徹底改變了微控制器的生態, 因為 Arduino 讓韌體開發變得非常容易, 使用類似 C 語法開發韌體, 易讀好懂, 開發環境 (IDE) 架設容易, 很容易就上手. 加上有各種的應用範例, 很多時候不需要從無到有開始編程, 調用範例稍微修改就可以達到我們要的應用. 真的是劃時代的進步. 我覺得應該可以徹底淘汰 8051 等微控器及其難用的開發方式. 自從瞭解了物聯網這個概念後, 便非常興奮的想知道如何透過網站和微處理器進行資料傳輸. 我的目標只是想要透過自己架設的服務器和微控制器連線, 我不想使用雲端平台如 Thingspeak, AWS IoT, Microsoft Azure, IBM Watson, MQTT 等. 我要裝置連線到我自己的網站, 然後我可以在前端開/關, 後端收到資料後重送命令到微處理器開 / 關 LED. 這就是我想要達成的目標. 我在 Udemy 買了兩堂課, Google 找到了一個 Github 專案, Youtueb 也找到一個相關的教學視頻, 雖然都不是我要的應用, 不過從程式碼中也可以一窺究竟. 以下是我在摸索的過程中得出來的結論.

# 開發過程 :

# ESP8266

  • 獲取網站的 fingerprints : Fingerprints
    • 例如 : 8E:68:78:DD:47:8F:03:BF:8B:51:DB:62:FE:5A:CB:6E:E9:20:82:A1
    • 移除冒號, 變成 8E 68 78 DD 47 8F 03 BF 8B 51 DB 62 FE 5A CB 6E E9 20 82 A1
    • 在程式標頭加入
      • const char fingerprint[] PROGMEM = "8E 68 78 DD 47 8F 03 BF 8B 51 DB 62 FE 5A CB 6E E9 20 82 A1";
    • 錯誤信息 class axTLS::wificlientsecure' has no member named 'setFingerprint'
      • 先暫時擱置 https, 使用本地 ip : 192.168.43.136 測試,
      • 程式碼 const char* host = "192.168.43.136";
