# AJAX

# 甚麼是 AJAX ?

廣義的說, 就是在不用重新整理網頁的情況下向後端伺服器撈資料

# 透過 XMLHttpRequest 物件跨瀏覽器撈資料

  • 開啟 Chrome console, 命令列輸入 const xhr = new XMLHttpRequest(), 按 Enter
  • 輸入 xhr, 按 Enter, 點擊左邊小箭頭會出現一長串資料
  • 注意 readyState: 0 意思是 你已經產生一個 XMLHttpRequest, 但是還沒有連結你要撈的資料

步驟

  1. 建立一個 xmlhttprequest : const xhr = new XMLHttpRequest()

  2. xhr.open('get', 'https://my-json-data.glitch.me/course.json', true)

    • 使用非同步方式讀取網址 https://my-json-data.glitch.me/course.json 的資料
    • xhr.open(格式, 要讀取的網址, 同步/非同步)
      • 格式 : get (讀取資料)、 post (傳送資料到伺服器)

  3. readyState: 1 : 你用了 open(), 但你還沒有把資料傳送過去

  4. xhr.send(null)

    • 我們只要讀取資料, 所以送出 null (空字串)

  5. readyState: 4 : 你撈到資料了, 數據已經完全接收到了


~ 完整程式碼 ~

const xhr = new XMLHttpRequest()

xhr.open('get', 'https://my-json-data.glitch.me/course.json', true)

xhr.send(null)

readyState 狀態說明

  • 0 : 你已經產生一個 XMLHttpRequest, 但是還沒有連結你要撈的資料
  • 1 : 你用了 open(), 但你還沒有把資料傳送過去
  • 2 : 偵測到你有用 send()
  • 3 : loading 中, 通常資料量大才有
  • 4 : 你撈到資料了, 數據已經完全接收到了

# AJAX 非同步觀念 (上)

xhr.open(,, 同步/非同步)

  • 非同步 (true) : 不會等資料傳回來, 就讓程式繼續往下跑,, 等到回傳才會自動回傳
  • 同步 (false) : 它會等資料傳回來, 才讓程式碼繼續往下跑

WARNING

xhr.open() 已經不需要加入 同步/非同步 的參數, 默認就是非同步傳送資料


# AJAX 非同步觀念 (下)

xhr.onload = () => {} : 處理撈到的資料

JSON.parse(物件) : 把物件轉換成字串格式

~警告視窗印出陣列第一個物件的 title 屬性值~

const xhr = new XMLHttpRequest()

xhr.open('get', 'https://my-json-data.glitch.me/course.json', true)
xhr.send(null)
xhr.onload = () => {
  const str = JSON.parse(xhr.responseText)
  alert(str[0].title)
}

# HTTP 狀態碼

Chrome console 輸入 xhr, 按 Enter, 點擊底下 XMLHttpRequest 左邊的小箭頭, 找到 status 屬性

  • 200 : 資料有正確回傳, 有撈到
  • 404 : 資料讀取錯誤, 沒有撈到

~如果狀態碼等於 200, 印出 title~

if (status == 200) {
  const str = JSON.parse(xhr.responseText)
  console.log(str[0].title)
}

<script>
const xhr = new XMLHttpRequest()

xhr.open('get', 'https://my-json-data.glitch.me/course.json')
xhr.send(null)
xhr.onload = () => {
  if (xhr.status == 200) {
    const str = JSON.parse(xhr.responseText)
    alert(`
      ${str[0].title}
      回傳成功 !
    `)
  }
}
</script>
